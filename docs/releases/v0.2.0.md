# ForgedTTC validator v0.2.0

This candidate advances the standalone `forge-validate` command and
`@forge/validate-wasm` facade to package version 0.2.0 and ModelSpec 2.2.0. It adds
explicit equipped slot alternatives without changing validator-report, replay, or
EnvSpec major versions.

ModelSpec 2.2 adds optional-shape `slots[].equippedVariantId` with a sovereign
validator invariant: every non-empty slot must name exactly one unique variant. Only
that alternative contributes inline geometry, catalog references, mass, simulation
inputs, collider checks, lockfile requirements, or BOM rows. The WASM artifact carries
stable source JSON Pointers so Studio edits and inspection state remain attached to
the physical source part when an alternative changes the flattened part table.

Migration from 2.1 is explicit and audit-recorded:

- a slot with one alternative is deterministically equipped and records
  `equip-single-variant-slots-v2.2`;
- a slot with multiple alternatives is not guessed. Set `equippedVariantId` to the
  intended variant, then rerun `forge-validate migrate ... --to current`.

The v0.2 worker/gateway artifact boundary also introduces the independently versioned
license export manifest 1.0. Manufacturing export jobs now fail closed without
compatible per-asset ledger evidence, require attribution manifests when applicable,
and substitute no-redistribution/view-only geometry with dimensioned envelopes,
datum ports, and BOM link-outs. Live OCCT artifact proof remains a sandbox milestone.

Gateway deployments also gain the SEC-002 platform-exclusion boundary. Generation
context, synchronous/streaming generation, course generation, and model edits reject
weapons, targeting, munition, and interdiction briefs before retrieval/provider or
mutation work. Structured refusals expose `SAFETY_PROHIBITED_BRIEF` and store only
hash/bucket/version/category/rule/surface/provider/archetype/optional-owner metadata;
raw refused prompts and provider keys are neither persisted nor echoed. Failure to
write this audit row fails closed.

Compatibility: validator report 1.x, replay 1.x plus the deprecated `replay.v1`
read alias, EnvSpec 1.x, and license export manifest 1.x. See
[`../COMPATIBILITY.md`](../COMPATIBILITY.md).

This release does not include FORGE Desktop, live providers, hosted operations, or
field-proven hardware claims. Hosted gateway deployments must apply the additive
`0015_generation_refusals.sql` migration; it has no backfill or object-storage impact.
On application rollback, retain the refusal audit table unless an explicit retention/
privacy decision authorizes an exported-and-backed-up purge. Standalone validator
rollback restores the v0.1.0 binary/package; documents saved with multiple ModelSpec
2.2 alternatives need an explicit 2.1-compatible conversion and must never silently
fall back to array order. Verify `SHA256SUMS` before use.
