## Authentication vocabulary

- `public`: no session is required.
- `optional-session`: anonymous and authenticated shapes are both supported.
- `optional-review`: public reads are supported; non-public curation filters need reviewer authority.
- `session`: an owner session is required and ownership is scoped server-side.
- `review-token`: a constant-time checked owner/reviewer bearer token is required.

Provider keys are accepted only by the dedicated generation header and are never part
of the OpenAPI request body, persistence, examples, or response schemas.

## Event behavior

Generation SSE emits one `start`, zero or more ordered `stage` records, then exactly
one terminal `complete` or `error`. Persisted job events are read in ascending
numeric `id` order and use `started`, `retry-scheduled`, `succeeded`, `failed`,
or `discarded`. A discarded event means a stale, cancelled, or lease-losing attempt
did not gain authority to materialize output. See the event catalog for fields and
stage values.
